# Научный клубок - R&D Knowledge Graph

Прототип для хакатона Норникеля: система собирает разрозненные научно-технические документы в проверяемую карту знаний R&D для горно-металлургической отрасли.

Главная боль: в R&D часто теряется не модель, а память организации. Команды не видят, что уже пробовали, где лежат источники, какие выводы подтверждены, где есть противоречия и какие темы слабо изучены. "Научный клубок" показывает это как граф фактов, источников и пробелов.

## Почему это не просто чат с PDF

- Система извлекает структурированные факты, а не только ищет похожие куски текста.
- Каждый факт связан с документом, chunk, опорной цитатой и confidence.
- Граф хранит сущности и связи: материалы, процессы, показатели, оборудование, документы, экспертов и выводы.
- Поиск поддерживает параметры: материал, процесс, условия, числовые диапазоны, год, география, отечественная/зарубежная практика, эксперт.
- Матрица пробелов показывает R&D-зоны риска: частые темы, между которыми в корпусе нет подтвержденной связки.
- Мини-валидация качества показывает ручную проверку 20+ фактов: верно / частично / ошибка.
- Аналитический ответ можно скачать как Markdown-отчет с источниками.

## Что реализовано в MVP

- `ingest.py` извлекает текст из PDF/DOCX в `extracted/`.
- `extract_facts.py` превращает текстовые фрагменты в `facts.jsonl` через OpenAI-compatible LLM.
- `build_graph.py` собирает граф знаний `graph.json` на NetworkX, добавляет `geo_scope` и Expert-узлы.
- `query_graph.py` выполняет поиск по фактам, условиям, годам, географии, типу практики, экспертам и числовым диапазонам.
- `gap_matrix.py` строит матрицу покрытия и HTML-версию `matrix.html`.
- `answer_synthesizer.py` собирает проверяемые аналитические ответы.
- `api.py` дает FastAPI-бэкенд (поиск, факты, соседи узла, матрица пробелов, аналитический ответ, экспорт JSON-LD) для React-интерфейса `ui/`.
- `ui/` — React + Vite интерфейс исследователя поверх API: запрос на естественном языке, фильтры, цепочки «материал → процесс → результат», источники с цитатами.
- `app.py` дает Streamlit-интерфейс для жюри и исследователей.
- `quality_validation.json` содержит мини-аудит ручной проверки фактов.
- Вкладка "Пополнение базы" позволяет загрузить PDF/DOCX/TXT или добавить факт вручную; новые данные сохраняются в карантин `expert_submissions.jsonl` и не смешиваются с основным графом без проверки.
- `synonyms.json` нормализует русские и базовые англоязычные термины: `electrowinning` -> `электроэкстракция`, `PGM` -> `МПГ`, `mine waters` -> `шахтные воды`.
- `units.py` конвертирует единицы измерения при числовых фильтрах: запрос по `500 мг/л` находит факт, записанный как `0.5 г/л`; `°C`/`K`/`°F`, `м3/ч`/`л/с`, `МПа`/`бар`/`атм` сопоставляются по размерности, несовместимые размерности не дают ложных совпадений.
- `export_jsonld.py` выгружает граф в JSON-LD с онтологическим `@context` и провенансом (источник, цитата, confidence, дата) по принципам FAIR; в API есть `GET /api/export/jsonld` (весь граф) и `POST /api/export/jsonld` (факты конкретного запроса для вставки в отчёт/ТЗ).

## Демо-сценарии для защиты

1. **Обессоливание воды**
   Показать найденные методы очистки и честно обозначить пробел по точной связке Ca/Mg/Na 200-300 мг/л и сухой остаток <=1000 мг/дм3.

2. **Циркуляция католита**
   Показать факты по электролиту, скорости потока и электроэкстракции, а также неполноту покрытия по схемам циркуляции.

3. **Au/Ag/МПГ: штейн и шлак**
   Показать междисциплинарный поиск по драгоценным металлам, шлакам, штейнам, годам и источникам.

4. **Закачка шахтных вод**
   Показать, что по очистке шахтных вод есть факты, а по закачке в глубокие горизонты текущий корпус дает пробел.

## Запуск демо

```powershell
pip install -r requirements.txt
python build_graph.py
python test_queries.py
.\run_demo.ps1
```

После запуска Streamlit откроет приложение на `http://localhost:8501`.

Альтернативный стек «FastAPI + React» (тот же граф, современный UI исследователя):

```powershell
uvicorn api:app --port 8000          # или .\run_api_server.cmd
cd ui; npm install; npm run dev       # Vite на http://localhost:5173
```

React-интерфейс ходит в API на `http://localhost:8000` и дает поиск, фильтры, цепочки связей, матрицу пробелов и экспорт JSON-LD.

Для повторного LLM-извлечения задайте ключ через переменные окружения, не сохраняйте его в коде:

```powershell
$env:API_KEY="your_openrouter_or_openai_compatible_key"
$env:BASE_URL="https://openrouter.ai/api/v1"
$env:MODEL_NAME="qwen/qwen3-32b"
python extract_facts.py --input demo_extracted --limit 25
```

Путь расширения на подготовленный корпус:

```powershell
python extract_facts.py --input extracted --limit 152
python build_graph.py
python test_queries.py
```

## Архитектура

```text
PDF/DOCX
  -> ingest.py
  -> extracted/*.txt
  -> extract_facts.py
  -> facts.jsonl
  -> build_graph.py
  -> graph.json + normalization_report.txt
  -> query_graph.py (+ units.py) + gap_matrix.py + answer_synthesizer.py + export_jsonld.py
  -> app.py (Streamlit)  |  api.py (FastAPI) -> ui/ (React + Vite)
```

## Проверяемость

Факты считаются автоматически извлеченными. Перед производственным применением они требуют экспертной проверки.

Текущая модель доверия:

- `verification_status=auto_extracted` означает, что факт найден автоматически.
- `confidence` отражает явность связи в исходном тексте, а не экспертное утверждение истины.
- `source_quote` и `source_file` нужны для быстрой ручной проверки спорных фактов.
- UI показывает дату сборки графа, покрытие метаданных и ограничения MVP.

## Как защищать проект за 60 секунд

> В R&D горно-металлургических исследований проблема часто не в отсутствии LLM, а в отсутствии общей памяти: что уже пробовали, где источники, какие выводы подтверждены, где противоречия и пробелы. Мы доказали ядро платформы: система берет PDF/DOCX, извлекает проверяемые факты, строит граф материалов, процессов, условий, результатов и экспертов, дает поиск по параметрам и показывает источники с цитатами. Главное отличие от RAG-чата - матрица пробелов: продукт не только отвечает, но и показывает, какие R&D-комбинации слабо покрыты корпусом. Production-слой требует графовой БД, экспертной верификации, ACL, аудита, версионности фактов и нормализации единиц.

> Дополнительно показан controlled review workflow: эксперт может загрузить новый документ или вручную добавить факт, система кладет результат в карантин на проверку и не смешивает его с утвержденным графом автоматически.

## Ограничения MVP

- Используется локальный NetworkX JSON, не production-графовая БД.
- Поиск использует легкую нормализацию слов, но не полноценный semantic parser.
- Числовые диапазоны поддерживаются с конверсией основных единиц (`units.py`); реестр покрывает частые размерности домена, но не все возможные единицы.
- `geo_scope` строится по базовым маркерам стран, площадок и организаций; production-слой должен использовать справочник географии.
- Мини-валидация качества покрывает демонстрационную выборку, а не весь корпус.
- Live-пополнение базы работает через карантин: загруженные файлы и LLM-результаты не попадают в основной `graph.json` без отдельной процедуры approve/merge.
- Роли, ACL, аудит пользователей, экспертная правка графа и версионирование фактов представлены как roadmap.
- Англо-русская мультиязычность покрыта словарем синонимов, но не полноценным машинным переводом запросов.
- Секреты для LLM API должны задаваться только через переменные окружения.
