# Научный клубок - R&D Knowledge Graph

Прототип для хакатона Норникеля: система собирает разрозненные научно-технические документы в проверяемую карту знаний R&D для горно-металлургической отрасли.

Главная боль: в R&D часто теряется не модель, а память организации. Команды не видят, что уже пробовали, где лежат источники, какие выводы подтверждены, где есть противоречия и какие темы слабо изучены. "Научный клубок" показывает это как граф фактов, источников и пробелов.

## Почему это не просто чат с PDF

- Система извлекает структурированные факты, а не только ищет похожие куски текста.
- Каждый факт связан с документом, chunk, опорной цитатой и confidence.
- Граф хранит сущности и связи: материалы, процессы, показатели, оборудование, документы, выводы.
- Поиск поддерживает параметры: материал, процесс, условия, числовые диапазоны, год, география, эксперт.
- Матрица пробелов показывает R&D-зоны риска: частые темы, между которыми в корпусе нет подтвержденной связки.
- Аналитический ответ можно скачать как Markdown-отчет с источниками.

## Что реализовано в MVP

- `ingest.py` извлекает текст из PDF/DOCX в `extracted/`.
- `extract_facts.py` превращает текстовые фрагменты в `facts.jsonl` через OpenAI-compatible LLM.
- `build_graph.py` собирает граф знаний `graph.json` на NetworkX.
- `query_graph.py` выполняет поиск по фактам, условиям, годам, географии, экспертам и числовым диапазонам.
- `gap_matrix.py` строит матрицу покрытия и HTML-версию `matrix.html`.
- `answer_synthesizer.py` собирает проверяемые аналитические ответы.
- `app.py` дает Streamlit-интерфейс для жюри и исследователей.
- `synonyms.json` нормализует русские и базовые англоязычные термины: `electrowinning` -> `электроэкстракция`, `PGM` -> `МПГ`, `mine waters` -> `шахтные воды`.

## Текущие метрики демо

- Демо-корпус: 25 отобранных документов из `demo_extracted/`.
- Подготовленный полный текстовый корпус: 152 txt-документа в `extracted/`.
- Граф: 4695 узлов и 12785 связей.
- Result-факты: 2524 связей типа `yields`.
- С условиями процесса: 2195 фактов.
- С числовым результатом: 1351 факт.
- С географией: 970 фактов.
- С экспертами или организациями: 1915 фактов.
- Confidence: high 1585, medium 900, low 39.

## Демо-сценарии для защиты

1. **Электроэкстракция никеля**
   Показать параметры, результаты, источники, цитаты, confidence, географию и связь с документами.

2. **Шахтные воды и очистка**
   Показать методы обработки, условия, результаты и скачивание Markdown-отчета.

3. **МПГ, штейн и шлак**
   Показать междисциплинарный поиск по металлургическим переделам и источникам.

4. **Матрица пробелов**
   Показать, что система не только отвечает, но и находит слабо покрытые комбинации "материал - процесс - показатель".

## Запуск демо

```powershell
pip install -r requirements.txt
python build_graph.py
python test_queries.py
.\run_demo.ps1
```

После запуска Streamlit откроет приложение на `http://localhost:8501`.

Для повторного LLM-извлечения задайте ключ через переменные окружения, не сохраняйте его в коде:

```powershell
$env:API_KEY="your_openrouter_or_openai_compatible_key"
$env:BASE_URL="https://openrouter.ai/api/v1"
$env:MODEL_NAME="qwen/qwen3-32b"
python extract_facts.py --input demo_extracted --limit 25
```

Путь расширения на подготовленный корпус:

```powershell
python extract_facts.py --input extracted --limit 152
python build_graph.py
python test_queries.py
```

## Архитектура

```text
PDF/DOCX
  -> ingest.py
  -> extracted/*.txt
  -> extract_facts.py
  -> facts.jsonl
  -> build_graph.py
  -> graph.json + normalization_report.txt
  -> query_graph.py + gap_matrix.py + answer_synthesizer.py + app.py
```

## Проверяемость

Факты считаются автоматически извлеченными. Перед производственным применением они требуют экспертной проверки.

Текущая модель доверия:

- `verification_status=auto_extracted` означает, что факт найден автоматически.
- `confidence` отражает явность связи в исходном тексте, а не экспертное утверждение истины.
- `source_quote` и `source_file` нужны для быстрой ручной проверки спорных фактов.
- UI показывает дату сборки графа, покрытие метаданных и ограничения MVP.

## Как защищать проект за 60 секунд

> В R&D горно-металлургических исследований проблема часто не в отсутствии LLM, а в отсутствии общей памяти: что уже пробовали, где источники, какие выводы подтверждены, где противоречия и пробелы. Мы сделали MVP карты знаний: система берет PDF/DOCX, извлекает проверяемые факты, строит граф материалов, процессов, условий и результатов, дает поиск по параметрам и показывает источники с цитатами. Главное отличие от RAG-чата - матрица пробелов: продукт не только отвечает, но и показывает, какие R&D-комбинации слабо покрыты корпусом. В хакатоне реализовано ядро платформы; production-контур расширяется Neo4j, экспертной верификацией, ролями, аудитом и версионностью фактов.

## Ограничения MVP

- Используется локальный NetworkX JSON, не production-графовая БД.
- Поиск использует легкую нормализацию слов, но не полноценный semantic parser.
- Числовые диапазоны поддерживаются, но без полной конверсии единиц.
- Роли, аудит пользователей, экспертная правка графа и версионирование фактов представлены как roadmap.
- Англо-русская мультиязычность покрыта словарем синонимов, но не полноценным машинным переводом запросов.
- Секреты для LLM API должны задаваться только через переменные окружения.
