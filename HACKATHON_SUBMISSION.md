# Хакатон Норникель - материалы сдачи

## Продукт

**Научный клубок** - MVP карты знаний R&D для горно-металлургических исследований. Продукт доказывает ядро платформы: разрозненные PDF/DOCX-документы превращаются в проверяемый граф фактов, источников, экспертов, географии, отечественной/зарубежной практики и пробелов.

## Боль

В R&D знания часто рассыпаны по отчетам, обзорам, презентациям и личным архивам. Из-за этого команды повторяют уже сделанные обзоры, не видят подтвержденные выводы, спорят из-за разных интерпретаций и медленно отвечают на технические вопросы.

## Что показывает MVP

- End-to-end пайплайн: документы -> текст -> факты -> граф -> поиск -> аналитический ответ.
- Проверяемость: каждый факт связан с источником, chunk, цитатой и confidence.
- Многопараметрический поиск: материал, процесс, условие, числовой диапазон, год, география, эксперт.
- Нормализованный признак практики: domestic / foreign / mixed / unknown.
- Expert-узлы: организации и авторы выделены из `lab_or_author` и связаны с документами, процессами и материалами.
- Матрица пробелов: система показывает, какие R&D-комбинации слабо покрыты корпусом.
- Мини-валидация качества: 20+ вручную проверенных фактов со статусом верно / частично / ошибка.
- Пополнение базы: эксперт может загрузить PDF/DOCX/TXT или добавить факт вручную; результат попадает в `expert_submissions.jsonl` на approve/reject, не ломая основной граф.
- Markdown-экспорт аналитического ответа для вставки в отчет или презентацию.

## Демо и запуск

```powershell
pip install -r requirements.txt
python build_graph.py
python test_queries.py
.\run_demo.ps1
```

Приложение запускается на `http://localhost:8501`.

## Артефакты

- `README.md` - описание продукта, запуск, соответствие ТЗ, ограничения.
- `app.py` - Streamlit-интерфейс для демо.
- `facts.jsonl` - извлеченные факты и выводы.
- `graph.json` - граф знаний для UI и поиска.
- `matrix.html` - статическая матрица покрытия.
- `normalization_report.txt` - отчет по нормализации и покрытию.
- `quality_validation.json` - мини-аудит качества извлечения фактов.
- `expert_submissions.jsonl` - runtime-очередь экспертных заявок, создается UI при загрузке файлов или ручном вводе фактов.
- `PITCH_SCRIPT.md` - сценарий защиты.
- `VIDEO_SCRIPT.md` - сценарий видео до 5 минут.
- `PRESENTATION_OUTLINE.md` - структура презентации.

## Ограничения MVP

MVP не заявляется как промышленная платформа. В хакатоне доказано ядро: ingest, extraction, normalization, graph build, query, analytics, UI, проверяемость источников, Expert-узлы, `geo_scope`, матрица пробелов и controlled review workflow для новых файлов. Production-слой должен добавить Neo4j/JanusGraph, экспертную верификацию, роли, ACL, аудит, версионность фактов, нормализацию единиц и enterprise-интеграции.

## Roadmap

1. Перенос графа в Neo4j/JanusGraph с Cypher/Gremlin-запросами.
2. Экспертная проверка фактов: review/approved/rejected, история правок.
3. Merge approved-заявок из `expert_submissions.jsonl` в основной граф с аудитом.
4. Расширение нормализации единиц и географии: справочники стран, площадок и конверсия единиц.
5. Роли и доступ: исследователь, аналитик, руководитель, администратор, внешний партнер.
6. Подключение патентов, справочников, каталогов экспериментов и новых доменов.
