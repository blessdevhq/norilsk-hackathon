# Хакатон Норникель - материалы сдачи

## Продукт

**Научный клубок** - MVP карты знаний R&D для горно-металлургических исследований. Продукт превращает разрозненные PDF/DOCX-документы в проверяемый граф фактов: материалы, процессы, условия, результаты, источники, эксперты, география и пробелы.

## Боль

В R&D знания часто рассыпаны по отчетам, обзорам, презентациям и личным архивам. Из-за этого команды повторяют уже сделанные обзоры, не видят подтвержденные выводы, спорят из-за разных интерпретаций и медленно отвечают на технические вопросы.

## Что показывает MVP

- End-to-end пайплайн: документы -> текст -> факты -> граф -> поиск -> аналитический ответ.
- Проверяемость: каждый факт связан с источником, chunk, цитатой и confidence.
- Многопараметрический поиск: материал, процесс, условие, числовой диапазон, год, география, эксперт.
- Матрица пробелов: система показывает, какие R&D-комбинации слабо покрыты корпусом.
- Markdown-экспорт аналитического ответа для вставки в отчет или презентацию.

## Демо и запуск

```powershell
pip install -r requirements.txt
python build_graph.py
python test_queries.py
.\run_demo.ps1
```

Приложение запускается на `http://localhost:8501`.

## Артефакты

- `README.md` - описание продукта, запуск, соответствие ТЗ, ограничения.
- `app.py` - Streamlit-интерфейс для демо.
- `facts.jsonl` - извлеченные факты и выводы.
- `graph.json` - граф знаний для UI и поиска.
- `matrix.html` - статическая матрица покрытия.
- `normalization_report.txt` - отчет по нормализации и покрытию.
- `PITCH_SCRIPT.md` - сценарий защиты.
- `VIDEO_SCRIPT.md` - сценарий видео до 5 минут.
- `PRESENTATION_OUTLINE.md` - структура презентации.

## Ограничения MVP

MVP не заявляется как промышленная платформа. В хакатоне реализовано ядро: ingest, extraction, normalization, graph build, query, analytics, UI. Production-слой должен добавить Neo4j/JanusGraph, экспертную верификацию, роли, аудит, версионность фактов, нормализацию единиц и enterprise-интеграции.

## Roadmap

1. Перенос графа в Neo4j/JanusGraph с Cypher/Gremlin-запросами.
2. Экспертная проверка фактов: review/approved/rejected, история правок.
3. Нормализация единиц и географии: domestic/foreign/mixed.
4. Роли и доступ: исследователь, аналитик, руководитель, администратор, внешний партнер.
5. Подключение патентов, справочников, каталогов экспериментов и новых доменов.
